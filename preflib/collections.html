Collections

PrefLib users tend to be interested in similar sets of files. On this page, we provide pre-compiled collections of files. If the set of files is not available here, use the preflibtools to generate your own collection.

Important: By using automatically-generated collections of files you loose the fine-grained understanding of the files in the collection (e.g., the provenance of the data). Keep that in mind as this can impact your results.

Data Type

Strict Order Complete (SOC)

Strict order complete files encode preferences in which all respondent submitted a strict ranking containing all the alternatives.

Download all files

Strict Order Incomplete (SOI)

Strict order incomplete files encode preferences in which all respondent submitted a strict ranking containing some of the alternatives.

Download all files

Tie Order Complete (TOC)

Strict order complete files encode preferences in which all respondent submitted a ranking containing all the alternatives where ties between alternatives are allowed.

Download all files

Tie Order Incomplete (TOI)

Strict order incomplete files encode preferences in which all respondent submitted a strict ranking containing some of the alternatives where ties between alternatives are allowed.

Download all files

Categorical (CAT)

Categorical files encode preferences in which all respondent classified some of the alternatives in categories. The categories are themselves globally ordered which defines preferences.

Download all files

Weighted Matching Data (WMD)

Weighted matching data files encode preferences that are seen as a weighted directed graphs, i.e., a collection of edges between the alternatives associated with a weight.

Download all files

Modification Type

Original Files

In original files the preferences appear as they were submitted.

Download all files

Induced Files

In induced files the preferences have been induced from another context but not modified: the format differs but not the content.

Download all files

Imbued Files

In imbued files the preferences have been induced from another context and additional (arbitrary) information has been added.

Download all files

Synthetic Files

In synthetic files the preferences have been generated artificially.

Download all files

Structural Properties

Approval Files

Approval files correspond to cases in which the preferences can be read as: each respondent approves of a given subset of alternatives. This includes:

  • Ordinal preferences with ties where each respondent submitted a single indifference class;
  • Ordinal preferences with ties where each respondent submitted two indifference classes containing all the alternatives;
  • Categorical preferences with 1 category;
  • Categorical preferences with 2 categories containing all the alternatives.
